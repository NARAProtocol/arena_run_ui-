## Design Context

### Users
Base-native NARA holders, board claimants, sponsors, and spectators who need to understand the race and act in a few seconds. The page should feel playable on first scan, not like a presentation deck.

### Brand Personality
Sharp, compact, credible. The interface should read like a serious on-chain game console built inside the NARA ecosystem, not a dark spectacle dashboard and not a generic DeFi admin panel.

### Aesthetic Direction
Light mode by default. Reuse the same visual family as the lockboard and lotto apps: warm paper background, mono typography, small labels, compact panels, thin borders, and restrained accent color. Avoid oversized tiles, giant display type, heavy dark surfaces, and theatrical command-deck styling.

### Design Principles
1. Put the playable actions first: join, move, sabotage, claim.
2. Keep numbers small, dense, and readable with tabular mono styling.
3. Use compact buttons and short labels instead of large decorative cards.
4. Keep prize, clock, sponsor, and locker context visible, but secondary to gameplay.
5. Prefer clarity and speed over spectacle. If a block does not help the user play, compress it.
